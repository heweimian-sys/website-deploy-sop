一、这个 SOP 是给谁用的

当你已经有一个网站项目，代码已经上传到 GitHub 后，可以用这个 SOP 完成上线。本指南基于新版 Cloudflare 中文界面制作，每一步都有截图标注。

适合的场景

  • 纯展示类网站（官网、活动页、个人主页）
  • 带搜索、登录等交互功能的网站
  • 工具页、落地页上线
  • 想让 AI 自动帮你部署到 Cloudflare（需要 API Token）

你需要准备的账号

账号干什么用的是否必须
GitHub 账号存放你的网站代码必须
Cloudflare 账号放你的网站，让用户能访问必须
Cloudflare D1存放后端数据，和 Pages 函数配合使用有后端数据时必须
Cloudflare API Token给 AI 的操作钥匙，让 AI 自动帮你部署想让 AI 自动部署时需要
代理工具（翻墙软件）注册和配置海外平台时需要必须（见第二章）
AI 工具（如 TRAE）遇到任何困难随时问 AI强烈建议
核心理念：你不需要懂代码。你只需要做两件事：跟着步骤点击操作，以及遇到不确定的地方把信息发给 AI 让它帮你判断。
什么是"前端"和"后端"？：简单理解——用户能看到和点击的页面叫"前端"，在背后处理数据、搜索、保存信息的程序叫"后端"。如果你的网站只有展示页面没有交互功能，那就只有前端，不需要后端。
什么是 API Token？：简单理解——API Token 就像一把钥匙，你把它交给 AI，AI 就能拿着这把钥匙帮你操作 Cloudflare（比如部署网站、管理数据库），不需要你每次都手动登录操作。详细申请方法见第七章。

二、网络环境说明（重要）

这是新手最容易忽略的一步：不同平台在国内的访问情况不同。不提前了解，会导致"明明操作对了但就是打不开"。

各平台在国内能不能打开？

平台国内状态用来干什么
Cloudflare Pages能直接打开放你的网站
DeepSeek能直接打开AI 服务（如果你的网站用到了）
GitHub时好时坏存代码
Cloudflare 操作后台建议开代理配置网站部署
Cloudflare D1建议开代理配置存放后端数据
Vercel打不开不要用这个

什么时候要开代理，什么时候不用？

你在做什么要开代理吗？说明
注册 GitHub / Cloudflare 账号要开注册时可能需要验证，不开代理可能卡住
在 Cloudflare 后台操作建议开开了更稳定，不容易加载失败
上传代码到 GitHub看情况有时候能传成功，传不上去就开代理
别人访问你的网站不用开网站放在 Cloudflare 上，国内直接能打开
记住这句话：你在配置和部署的时候需要开代理，但你的用户访问网站时不需要开代理。

上传代码到 GitHub 总是失败？

如果你在上传代码到 GitHub 时经常失败，可能是网络问题。不要自己折腾网络设置，直接问 AI：

发给 AI 的提示词
我在上传代码到 GitHub 时总是失败，提示连接超时。 我的电脑用的是 Windows 系统，代理软件是【填你的代理软件名称，比如 Clash / V2Ray 等】。 请帮我： 1. 判断是不是代理没配置好 2. 如果需要配置，请告诉我具体怎么操作（一步一步来） 3. 配置好了之后怎么测试是否成功

三、部署前准备

在开始部署前，确认以下内容：

  1. 已注册并登录 Cloudflare 账号（注册时需要开代理）
  2. 已有 GitHub 账号，网站代码已经上传到 GitHub 仓库
  3. 如果你的网站有搜索、登录等交互功能，还需要准备好后端代码的 GitHub 仓库
部署时需要填一些配置信息，但你不需要自己判断怎么填：Cloudflare 会问你一些问题（比如"你的项目用什么技术""构建命令是什么"），这些你不需要懂。把你的 GitHub 仓库地址发给 AI，AI 会告诉你每个空填什么。
发给 AI 的提示词
我要把网站部署到 Cloudflare Pages，这是我的 GitHub 仓库地址：【填你的仓库地址】 请你帮我看看： 1. 这个网站是什么类型的？ 2. Cloudflare 会问我的"构建命令"应该填什么？ 3. "输出目录"应该填什么？ 4. 有没有需要提前准备的东西？ 请用大白话告诉我，我完全不懂代码。

四、调整界面语言为中文

Cloudflare 默认界面是英文的，对于新手来说不太友好。建议第一步就把界面改成中文，这样后续操作的每个按钮你都能看懂。

为什么要改中文？：Cloudflare 后台的按钮和菜单本来就有中文翻译，改成中文后，"Create application"会变成"创建应用程序"，"Deployments"会变成"部署"，操作起来轻松很多。本 SOP 后续所有步骤都基于中文界面进行。
1

登录 Cloudflare 后台

打开 https://dash.cloudflare.com/（建议开代理），登录后进入控制台首页。

2

点击右上角头像图标

在页面最右上角，找到你的头像/用户图标（一个小人形图标），点击它会弹出一个下拉菜单。在菜单中找到「语言」选项。

3

选择「语言」→「简体中文」

在下拉菜单中找到「语言」选项（如果当前是英文，显示为 Language），鼠标移上去会自动展开一个子菜单，列出所有支持的语言。选择「简体中文」即可。

设置完成！：选择后界面会立刻变成中文，不需要刷新页面。从此以后，Cloudflare 的所有菜单和按钮都会显示中文，操作起来轻松多了。接下来按第五章开始部署前端。

五、前端部署到 Cloudflare Pages

这是核心步骤。Cloudflare Pages 是一个免费的网站托管平台，放在上面的网站国内用户可以直接访问。本章基于新版 Cloudflare 中文界面制作。

1

进入 Workers 和 Pages

在左侧菜单栏中，找到「计算」分类，点击展开后选择「Workers 和 Pages」。这里是你管理所有部署项目的地方。

Workers 和 Pages 是什么：就是 Cloudflare 里"管理网站"的页面。进去之后可以创建新的网站项目。新版 Cloudflare 把它放在了"计算"分类下面。
2

点击「创建应用程序」

在 Workers 和 Pages 页面右上角，点击蓝色的「创建应用程序」按钮。

3

选择部署 Pages

新版的创建页面标题是"Ship something new"（发布新东西），会显示多个选项。点击第一个选项「Continue with GitHub」（用 GitHub 继续），直接进入 GitHub 仓库选择流程。

注意：如果你的界面没有看到"Continue with GitHub"选项，可以向下滚动到页面底部，找到「想要部署 Pages？开始使用」链接，点击它也可以进入 Pages 部署流程。
4

选择从 Git 导入

进入 Pages 创建页面后，你会看到两个选项。选择上方的「导入现有 Git 存储库」卡片，点击其右侧的「开始使用」按钮。这样可以从你的 GitHub 仓库直接导入代码。以后你更新了 GitHub 上的代码，Cloudflare 会自动帮你重新部署。

5

连接 GitHub 并选择仓库

首次使用需要授权 Cloudflare 访问你的 GitHub。点击「连接到 Git」，按提示完成 GitHub 授权。然后在列表中选择你要部署的仓库。仓库太多可以在搜索框输入名称搜索。

找不到仓库怎么办？：如果列表里看不到你的仓库，说明 Cloudflare 还没获得该仓库的访问权限。点击页面底部的授权链接，跳转到 GitHub 设置页面，勾选目标仓库 → 保存 → 回到 Cloudflare 刷新即可。
6

填写配置信息（不知道怎么填就问 AI）

选择仓库后会进入配置页面，需要填写项目名称、生产分支、构建命令、输出目录等。这里有术语你不需要自己判断怎么填。

不要自己瞎填！：把你 GitHub 仓库地址发给 AI，AI 会告诉你每个空填什么。填错了会导致部署失败。
发给 AI 的提示词
我正在 Cloudflare Pages 上部署网站，现在需要填写配置信息。 我的 GitHub 仓库是：【填你的仓库地址】 请告诉我： 1. "项目名称"（Project name）填什么？ 2. "生产分支"（Production branch）填什么？ 3. "构建命令"（Build command）填什么？ 4. "构建输出目录"（Build output directory）填什么？ 5. "环境变量"（Environment variables）需要填吗？填什么？ 请一个一个告诉我，用大白话说。
7

点击「保存并部署」

配置完成后，点击页面底部的「保存并部署」按钮。Cloudflare 会开始自动构建你的网站。

8

等待部署完成

部署过程中会显示构建过程（一堆日志，看不懂没关系）。如果部署成功，会出现成功提示，并展示一个 .pages.dev 网址。

如果部署失败了：不要慌，也不要反复点击重新部署。把构建日志里红色的报错信息复制发给 AI，让 AI 帮你找原因。
9

找到正式网站地址

部署成功后，在项目详情页的「部署」标签页中，可以看到你的网站地址：

https://你的项目名.pages.dev

这个地址就是正式访问地址，可以分享给任何人。

两种地址的区别：正式地址 xxx.pages.dev 是对外分享用的；还有一种带随机前缀的地址（比如 4601e4a2.xxx.pages.dev）是某次部署的预览链接，一般不用管。对外分享用正式地址。

六、后端部署到 Cloudflare D1

这一版后端统一放在 Cloudflare：后端数据存储使用 Cloudflare D1，并通过 Pages 函数的绑定（Bindings）连接到当前 Pages 项目。这样前端和后端都在 Cloudflare 体系内，部署链路更短。

你要完成的事：先创建一个 D1 数据库，再回到当前 Pages 项目，在设置 → 函数 → 绑定中添加 D1 数据库绑定。

操作步骤

1

进入 D1 数据库页面

在 Cloudflare 控制台左侧菜单中，找到「存储和数据库」分类，点击「D1 SQLite 数据库」进入 D1 管理页面。这里可以看到你已有的数据库列表。

2

点击三点菜单 → 创建数据库

在 D1 数据库页面，点击右上角的三点菜单（⋯），在弹出的菜单中选择「创建数据库」。如果页面直接显示了「创建数据库」按钮，直接点击即可。

3

填写数据库名称

输入数据库名称（建议用项目名或业务名，比如 zhixin-logs），位置保持默认（让 Cloudflare 自动选择），点击创建。

4

确认数据库创建成功

创建完成后，D1 数据库列表里会出现新的数据库记录。点击数据库名称进入详情页，可以看到数据库 ID，后面绑定时可能需要用到。

5

回到 Pages 项目设置

回到 Workers 和 Pages 列表，点击你的网站项目进入详情页。切换到「设置」标签页，找到「函数」区域下的「绑定」。

6

添加 D1 数据库绑定

在绑定区域点击「添加」，选择绑定类型为「D1 数据库」（不要选 KV、R2 或其他类型）。

  • 变量名：填写后端代码约定的名称（问 AI 你的代码里用的是什么名称）
  • D1 数据库：选择刚创建的数据库

填写完成后点击「保存」。

变量名很重要！：变量名必须和后端函数代码里读取的名称一致，否则网站功能会报错。不确定的话，把后端代码发给 AI 问它变量名是什么。
7

重新部署

保存绑定配置后，需要重新部署一次 Pages 项目才能生效。在「部署」页面点击「重试部署」或推送一次代码到 GitHub 触发自动部署。

怎么找到「重试部署」：在部署列表中，每条部署记录右侧都有一个「更多操作」（⋯）按钮，点击后会展开菜单，其中包含「重试部署」选项。也可以直接推送一次代码到 GitHub 来触发自动部署，效果一样。

绑定完成后检查

检查项正确结果
D1 数据库Cloudflare 控制台里能看到刚创建的数据库
Pages 绑定Pages 项目的「设置 → 函数 → 绑定」里已添加 D1 数据库
变量名绑定变量名必须和后端函数代码里读取的名称一致
重新部署保存配置后，重新部署或触发一次 Pages 构建

七、申请 Cloudflare API Token

API Token 是什么？用大白话说：API Token 就像是给 AI 的一把钥匙。正常情况下，部署网站需要你手动登录 Cloudflare 操作。但如果你想让 AI 自动帮你部署，就需要给它一个 API Token——AI 拿着这把"钥匙"，就能直接操作你的 Cloudflare 账户，比如部署网站、管理数据库，不需要你每次都手动登录点击。

简单理解：API Token = 给 AI 的 Cloudflare 操作权限
为什么需要 API Token？：如果你经常需要更新网站，每次都手动登录 Cloudflare 操作很麻烦。有了 API Token，你可以直接告诉 AI"帮我把 GitHub 仓库的最新代码部署到 Cloudflare"，AI 就能自动完成，省时省力。它还能让 AI 帮你管理数据库、查看部署日志等。

申请前的准备

  • 已登录 Cloudflare 账号
  • 建议开代理，确保 Cloudflare 后台稳定加载
  • 准备好一个记事本，用来保存 Token（创建后只显示一次！）

操作步骤

1

进入 API 令牌页面

在 Cloudflare 控制台左侧菜单中，找到「帐户 API 令牌」（通常在菜单底部的「管理帐户」分类下）。也可以通过点击右上角头像 → 在菜单中找到相关入口。

2

点击「创建令牌」

在 API 令牌页面，点击右上角的「创建令牌」按钮，进入令牌配置页面。

3

配置令牌信息

在令牌创建页面，需要填写以下信息：

  • 令牌名称：起一个你认得出的名字，比如"给 AI 部署用"
  • 权限策略：选择「自定义」（Custom）
  • 权限：选择以下权限（让 AI 能部署网站和管理数据库）：
    • 帐户 → Cloudflare Pages → 编辑
    • 帐户 → D1 → 编辑
  • 范围：选择「整个帐户」（这样 AI 可以操作你帐户下的所有项目）
4

审核令牌 → 创建

配置完成后，点击页面底部的「审核令牌」按钮（新版 Cloudflare 界面的按钮名称），进入令牌摘要确认页面。确认权限设置正确后，点击「创建令牌」。

5

复制并保存 Token（最重要的一步！）

⚠️ Token 只显示这一次！：创建成功后，页面会显示一串很长的字符，这就是你的 API Token。这串字符只会显示这一次，关闭页面后就再也看不到了！请立即复制并保存到安全的地方（比如密码管理器、记事本等）。
Token 长什么样？：Token 是一串以字母 c 开头的长字符串，类似 c5f3a2b8e9d1...（约40-50个字符）。如果你看到的字符串不是 c 开头的，可能复制错了内容。
Token 就像密码：不要把它发到公开的地方（比如微信群、论坛），任何拿到 Token 的人都能操作你的 Cloudflare 帐户。只在你信任的 AI 工具里使用。
为什么这一步没有截图？：Token 创建成功页面会显示真实的 API 密钥，属于敏感信息，不适合截图保存。你实际操作到这一步时，按照上面的说明复制保存即可。
6

如何使用 API Token

把 API Token 告诉 AI 助手，AI 就可以通过它帮你自动部署网站到 Cloudflare。

发给 AI 的提示词
这是我的 Cloudflare API Token：【粘贴你的 Token】 请帮我完成以下操作： 1. 把我的 GitHub 仓库【填仓库地址】部署到 Cloudflare Pages 2. 项目名称用：【填项目名】 3. 部署完成后告诉我网站地址 如果你需要其他信息（比如构建命令、输出目录），请先查看我的仓库代码，然后自己判断怎么填。
另一个场景：让 AI 更新已部署的网站
我的 Cloudflare API Token 是：【粘贴你的 Token】 我的 GitHub 仓库【填仓库地址】刚才更新了代码， 请帮我触发 Cloudflare Pages 重新部署，部署完成后告诉我结果。

Token 管理小贴士

八、部署后检查事项

网站上线后，不要只看 Cloudflare 显示成功，还需要自己打开网站检查一遍。

检查清单

  1. 打开正式网站地址，确认页面能正常打开
  2. 检查首页内容是否显示正常（文字、图片有没有错位）
  3. 检查按钮、链接、图片是否能正常点击和显示
  4. 用手机打开网站地址试试（很重要！）
  5. 如果网站有搜索功能，试试搜索能不能用
  6. 如果网站有表单，试试能不能提交
最重要的一步：用手机打开网站地址试试！电脑能打开不代表手机也能打开。如果手机打不开或有错位，截图发给 AI 让它帮你调整。
如果网站有问题：不要自己猜原因。描述清楚问题（比如"首页图片不显示""搜索点了没反应"），截图或录屏发给 AI，AI 能帮你判断是哪里出了问题。

九、遇到问题怎么办

万能解决方法：不管遇到什么问题，都是同一套流程——截图或复制报错信息 → 发给 AI → AI 告诉你怎么改 → 改完重新部署。下面是一些常见问题和对应的提示词。

问题 1：在 Cloudflare 找不到 GitHub 仓库

原因：Cloudflare 没有获得你 GitHub 仓库的访问权限。

解决：点击页面底部的授权链接，跳转到 GitHub 重新勾选目标仓库授权，保存后回到 Cloudflare 刷新。

问题 2：部署失败了（显示红色）

原因：可能是配置填错了、代码有问题、或者依赖文件冲突。

解决：不要反复点击重新部署！把构建日志里的红色报错信息复制发给 AI：

发给 AI 的提示词
我的网站在 Cloudflare Pages 部署失败了。 GitHub 仓库是：【填你的仓库地址】 这是报错信息： 【粘贴构建日志里红色的报错内容】 请帮我： 1. 判断是什么原因 2. 应该怎么修改 3. 修改后怎么重新部署

问题 3：部署成功但页面是空白的

原因：通常是配置里的"输出目录"填错了。

解决：把你的 GitHub 仓库地址发给 AI，问 AI"输出目录应该填什么"，然后去 Cloudflare 修改配置重新部署。

问题 4：网站能打开但搜索/功能用不了

原因：可能是 Pages 函数没正常运行，或者 D1 绑定变量名和代码不一致。

解决：

  1. 去 Cloudflare Pages 项目里检查 设置 → 函数 → 绑定
  2. 确认 D1 数据库已绑定，并且变量名和后端代码一致
  3. 保存配置后重新部署，再把浏览器报错和函数日志发给 AI

问题 5：Cloudflare 显示 522 错误

原因：可能是网站还在构建中，或者构建失败了。

解决：去 Cloudflare 控制台「部署」页面看看构建状态。如果显示红色（失败），按问题 2 的方法处理。

问题 6：API Token 不起作用

原因：可能是权限不够，或者 Token 已过期/被删除。

解决：回到 API 令牌页面检查 Token 是否还存在。如果权限不够，删除旧 Token 重新创建一个权限更大的（确保包含 Pages 编辑和 D1 编辑权限）。

遇到问题记住三个"不要"：不要慌、不要反复点击部署、不要自己瞎改配置。先把报错信息发给 AI，AI 帮你判断完再改。

十、给 AI 的提问模板

整个部署过程中，遇到任何不确定的地方，都可以用以下模板问 AI。记住一个原则：信息越详细，AI 回答越准。

模板 A：刚开始部署，不知道怎么填配置
我要把网站部署到 Cloudflare Pages。 GitHub 仓库地址是：【填你的仓库地址】 请你帮我判断： 1. 这个网站是什么类型的？ 2. Cloudflare 的"构建命令"填什么？ 3. "输出目录"填什么？ 4. 需要填"环境变量"吗？填什么？ 5. 如果部署失败，我先检查哪里？ 请用大白话告诉我，我完全不懂代码。
模板 B：部署/运行报错了
我在部署/使用网站时遇到了问题。 GitHub 仓库：【填你的仓库地址】 网站地址：【填你的网站地址】（如果有的话） 这是报错信息： 【粘贴报错日志，或描述问题】 请帮我： 1. 判断是什么原因 2. 应该怎么修改 3. 如果需要你帮忙改代码，请直接帮我改好并推送 我不懂代码，请直接帮我解决。
模板 C：用 API Token 让 AI 自动部署
这是我的 Cloudflare API Token：【粘贴你的 Token】 我的 GitHub 仓库是：【填你的仓库地址】 请帮我： 1. 把这个仓库部署到 Cloudflare Pages 2. 构建命令和输出目录请你自己查看仓库代码判断 3. 部署完成后告诉我网站地址 4. 如果需要后端 D1 数据库，也帮我创建并绑定 如果有任何问题请告诉我，我会提供更多信息。
模板 D：网站打开了但功能不正常
我的网站已经部署到 Cloudflare 了， 地址是：【填你的网站地址】 但是遇到以下问题： 【描述问题，比如：搜索功能用不了 / 页面显示不全 / 图片不显示 / 手机上打不开】 请你帮我排查原因，告诉我应该怎么解决。 如果有截图我发给你看。
提问小技巧：给 AI 的信息越详细越好——仓库地址、报错日志、截图、你操作到哪一步了、你期望的结果是什么。信息越全，AI 回答越准。

十一、后续维护

更新网站内容

如果你修改了网站代码（或让 AI 帮你改了），只需要把新代码推送到 GitHub，Cloudflare 就会自动重新部署。不需要手动操作。

怎么推送？：如果你用 AI 编程工具（如 TRAE），直接告诉 AI "帮我推送到 GitHub"就行。推送后 Cloudflare 通常 2-3 分钟后自动更新。
发给 AI 的提示词
我已经改好了网站代码，请帮我推送到 GitHub。 推送后帮我看看 Cloudflare 有没有自动开始重新构建。

用 API Token 让 AI 自动更新

如果你已经有 API Token，可以直接让 AI 帮你完成从代码更新到部署的全过程：

发给 AI 的提示词
我的 Cloudflare API Token 是：【粘贴你的 Token】 我的 GitHub 仓库是：【填你的仓库地址】 我刚才更新了代码，请帮我： 1. 推送代码到 GitHub 2. 确认 Cloudflare Pages 是否自动开始重新构建 3. 构建完成后告诉我结果

查看网站状态

  • Cloudflare：控制台 → Workers 和 Pages → 点你的项目 → 部署 看每次部署是否成功（绿色=成功，红色=失败）
  • Cloudflare 函数：Pages 项目 → 函数 查看函数日志和构建状态

查看网站访问数据

Cloudflare 自带免费的数据统计功能：

  • 控制台 → 点你的项目 → 分析 标签
  • 可以看到：访问次数、访客数量、流量来源、访问者国家分布等

回退到旧版本

如果新版本有问题，可以回退到之前能用的版本：

  • Cloudflare：部署页面 → 找到之前显示绿色的那次构建 → 点击「提升到生产环境」
找不到按钮？：截图 Cloudflare 页面发给 AI，告诉 AI"我想回退到旧版本"，AI 会指导你点哪个按钮。

API Token 安全管理

  • 定期检查 API 令牌页面，不需要的 Token 及时删除
  • 如果 Token 怀疑泄露，立即删除并重新创建
  • 不要在公开场合（微信群、论坛）分享 Token
  • 可以为不同用途创建不同 Token，方便管理

十二、流程总结

第一步：设置中文界面（建议首先做）

  1. 登录 Cloudflare → 点击右上角头像 → 「语言」→ 选择「简体中文」
  2. 界面立即切换为中文，后续操作更轻松

第二步：前端部署（必须）

  1. 左侧菜单「计算」→「Workers 和 Pages」→ 点「创建应用程序」
  2. 点击「Continue with GitHub」（或滚动到底部点「想要部署 Pages？开始使用」）
  3. 选「导入现有 Git 存储库」→ 连接 GitHub → 选择仓库
  4. 填写配置信息（不知道怎么填就问 AI）
  5. 点「保存并部署」→ 等待构建完成
  6. 复制 xxx.pages.dev 网站地址

第三步：后端部署（有搜索/登录等功能时需要）

  1. 左侧菜单「存储和数据库」→「D1 SQLite 数据库」
  2. 点击三点菜单（⋯）→「创建数据库」→ 输入名称 → 创建
  3. 回到 Pages 项目 → 设置 → 函数 → 绑定
  4. 添加 D1 数据库绑定，变量名和代码保持一致
  5. 保存配置后重新部署 Pages 项目

第四步：申请 API Token（想让 AI 自动部署时需要）

  1. 左侧菜单找到「帐户 API 令牌」
  2. 点击「创建令牌」→ 填写名称 → 选择权限（Pages 编辑 + D1 编辑）
  3. 范围选「整个帐户」→ 创建
  4. 立即复制保存 Token（只显示一次！）
  5. 把 Token 给 AI，AI 就能帮你自动部署

三个重要地址（记好）

名称地址格式用来干什么
网站地址https://xxx.pages.dev分享给用户访问
D1 绑定名DB 或你的代码约定名称告诉 AI，用来核对函数代码
API Token一串很长的字符给 AI 用，让它自动部署

记住这些原则

  1. 先改中文界面，操作起来更轻松
  2. 网站部署选 Pages，不要选 Worker
  3. 找不到 GitHub 仓库，去检查授权
  4. 不知道配置怎么填，不要瞎填，问 AI
  5. 部署成功不等于网站没问题，要自己打开检查
  6. 对外分享用 xxx.pages.dev 正式地址
  7. 遇到失败，先复制报错信息问 AI，不要反复点击部署
  8. API Token 就像密码，只给信任的 AI 工具，不要公开
  9. 你不是一个人在战斗：技术的事交给 AI，你负责点击和判断
